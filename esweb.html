WeaS

Weas stands for weather & security for our user's concern, which takes up a little space beside the door but can offer meaningful functions for your life.

People likely tend to forget to check about weather conditions for the day and thus can lead to embarrassments like stuck in the rain and dress less warm than needed. The thing is we don't have the habit to check weather forecast deliberately, but what we do is putting on sneakers, boots or leather shoes by the door when we heading out. We want to utilize the several seconds we spent on the shoes to inform our user about today's weather condition. The obvious red LED light on the weather forecast indicates today is or will be raining/snowing which could suggest our user check the LCD screen for more detailed info about today's weather. If the weather gonna be great today the peaceful green LED light will replace the red light.

Sometimes when we are out of the house, we would probably worry about our possession's safety. Our WeaS house guardian function will take care of the security concern for you. There are two modes for the house guardian, safe mode & security mode. In the security mode, if someone wants to break into the room without your permission, WeaS will detect that person, trigger the alarm and inform you via text messages. You can switch these two modes using the select buttons we provide you. What if we forget to set security mode when we leave the house? We can always set the mode using text messages and if setting successful you will get a confirmation text message from WeaS. One thing to notice: if the alarm has been triggered, these select buttons will be disabled so the alarm won't stop until you text WeaS.

Weather Forcast Implementation

The weather forecast mode requires a very small portion of hardware and the main idea behind this part is people don't usually check the weather on their phones before leaving home and this could make one come back home again to fetch an umbrella or even stuck in a rain etc. If we can utilize the IoT device to retrieve weather info and display them to our user just beside the door, then once our user heads out, he/she cannot miss the essential weather information and can avoid so many embarrassing situations due to the neglect of weather information.

The general working way can be described as follows: we receive weather information from the internet. This information shall be updated every hour and can be view by users. If the weather forecast indicates bad weather in the upcoming day/night, the red warning light will lit up and on the contrast, the green 'safe' light will list up. What we will show to our users are current weather, current temperature, daylight forecast and nightlight forecast. Since we have so much information to display and the LCD is quite small due to funding constraint, we use a button to choose what content to display on the LCD and use LED lights to indicate the user if he/she should check more detailed info by pushing the button (will be referred as the select button).

In order to interact with the internet, we need a WiFi mode which embedded in our microcontroller. The Adafruit feather HUZZAH board served the purpose very well. We can easily use the board to make HTTP request which is exactly what we need. We found a weather information API called Accurate Weather, that allows us to make 50 free HTTP request per day. Get HTTP request will make the API respond to our request. After we extract useful info from the API respond by processing the string, we should then display it on the LCD screen. And if the weather info indicates there is/will be a rain or snow today, we will lit the red light warning users to check more by pushing the select button. To enable the select button, we use the rising outer interruption caused by the level rising when users push the button. Once we receive the button push signal, the display should change to the next state.

House Guardian Implementation

In this part, we first need Arduino to control the whole process and watch over the interrupts and then we also need SIM900 which uses AT commands to communicate our user’s phone. There are two modes: safe mode and security mode. We could use either button in the product or user’s phone to switch mode but if in the security mode, someone breaks in then only the user’s phone could disable the alarm and switch it to safe mode.

In safe mode, we would show the real-time (real-time would update per minute) in the LCD screen, the green LED would light up and we enable the safe mode button and disable the security mode button. In security mode, we would also show the real time in the LCD screen, the red LED would light up and we enable the security mode button and disable the safe mode button and the distance sensor would enable to detect whether someone breaks in.

And we could separate the total progress into several states.
I. Initialization state: In this state, we initialize the button interrupts RTC LCD and register the online status for SIM900 to connect with the net.

II. Safe Mode state: In this state, we would show the real time in the LCD screen, the green LED would light up and we enable the safe mode button and disable the security mode button. In this mode, we could either press the security button or use user’s phone to send “securitymode” message to switch the mode to security mode.

III. Security Mode state: In this state, we would also show the real time in the LCD screen, the red LED would light up and we enable the security mode button and disable the safe mode button and the distance sensor would enable to detect whether someone breaks in. In this mode, we could either press the safe button or use user’s phone to send “safemode” message to switch the mode to security mode. If someone breaks in, then our product would send a message to the user,” warning someone breaks in!!!”, the beeper would alarm and the red LED would flash per half second and in this situation, we would disable the safe mode button in the circuit and only the user’s phone switch the mode to safe mode by sending the “safemode” message.
